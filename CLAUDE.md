## Project Overview

A data pipeline for a movie recommendation/analytics system that fetches data from the TMDB API, loads it into Snowflake, and transforms it through a medallion architecture (Bronze → Silver → Gold) using dbt.

The first pipeline (`tmdb_pipeline`) is fully operational — ingesting genres from TMDB, running dbt transformations, and executing dbt tests end-to-end.

## Architecture

### Stack
- **Orchestration:** Apache Airflow 3.0.1 (CeleryExecutor with Redis broker)
- **Data Warehouse:** Snowflake (`DB_FIND_A_MOVIE` database)
- **Transformation:** dbt-core with dbt-snowflake adapter
- **Metadata DB:** PostgreSQL 13 (persisted via Docker volume `find_a_movie_postgres-db-volume`)

### Key Directories
```
airflow/dags/

├── database_connections/              — Snowflake connection utilities
```

### DAG Structure Pattern
Each DAG follows this flow:
```

```
- `dbt_run` — `cd /opt/airflow/dbt && dbt run --target prod`
- `dbt_test` — `cd /opt/airflow/dbt && dbt test --target prod` with `trigger_rule=ALL_DONE`

## Ingestion Patterns

### Bronze layer — Airflow tasks
- **Append only:** never DELETE or TRUNCATE in ingestion tasks. Each run adds new rows.
- **No primary keys** on Bronze tables — the same source ID will appear multiple times across runs.
- **Mandatory audit columns** on every Bronze table:
  - `endpoint` VARCHAR — API path called (e.g. `/genre/movie/list`)
  - `raw_json` VARIANT — full API response as-is, parsed via `PARSE_JSON()`
  - `ingested_at` TIMESTAMP_NTZ — defaults to `CURRENT_TIMESTAMP()`
- **raw_json content:** store the full API response on every row (not per-record JSON)

### Snowflake INSERT with VARIANT (PARSE_JSON)
Never use `executemany` with `PARSE_JSON(%s)` inside `VALUES` — the Snowflake connector does not support function calls there. Always use the `SELECT ... FROM VALUES` pattern:
```python
insert_sql = """
    INSERT INTO table (col1, col2, raw_json)
    SELECT column1, column2, PARSE_JSON(column3)
    FROM VALUES {}
""".format(",".join(["(%s, %s, %s)"] * len(rows)))
cursor.execute(insert_sql, [v for row in rows for v in row])
```

### Docker Compose — environment variables
Variables defined in `.env` are used for YAML substitution only (`${VAR}`). They are **not** automatically injected into containers. Any variable needed at runtime must be explicitly declared in the `environment` section:
```yaml
SNOWFLAKE_PRIVATE_KEY_PATH: ${SNOWFLAKE_PRIVATE_KEY_PATH}
```

## dbt Conventions

### Model naming
- Bronze: `base_bronze_<TABLE_NAME>.sql` — generated by dbt plugin from source
- Silver: `silver_<entity>.sql` — deduplicated, cleaned, includes `endpoint` and `ingested_at`
- Gold: `dim_<entity>.sql` / `fct_<entity>.sql` — clean business columns only, no audit columns

### Config block (required on every model)
```sql
{{ config(
    materialized='table',  -- or 'view' for bronze
    schema='SILVER',       -- BRONZE, SILVER, or GOLD
    tags=['silver', 'tmdb', 'entity']
) }}
```

### Silver deduplication pattern
```sql
qualify row_number() over (partition by <id_col> order by INGESTED_AT desc) = 1
```

### Tests (yml files)
Every model must have a `.yml` file with at minimum:
- `unique` + `not_null` on the primary key
- `not_null` on all other columns

## External APIs

### TMDB API
Documentation is at `docs/TMDB Api Reference.md`. Read this file whenever working with
TMDB API integrations, endpoints, authentication, or data models.

**Account setup:**
1. Create a free account at `https://www.themoviedb.org/signup`
2. Go to Settings → API → Request an API Key → Developer
3. Fill in app name (`find_a_movie`), URL (`http://localhost`), type (`Desktop Application`)
4. After approval, copy the **API Read Access Token** (long `eyJ...` JWT — not the short API Key)

**Airflow Variable:**
Add the token to Airflow before running any TMDB DAG:
- URL: `http://localhost:8080` → Admin → Variables → `+`
- Key: `TMDB_API_READ_ACCESS_TOKEN`
- Value: paste the full `eyJ...` token

## Environment Setup

Template files for setup (copy and fill in credentials):
- `.env.example` → `.env`
- `dbt/profiles.yml.example` → `dbt/profiles.yml`

Copy `.env.example` to `.env` before running Docker:
```bash
cp .env.example .env
```

The `.env` file must contain:
```

AIRFLOW_PROJ_DIR=<absolute path to project root>
SNOWFLAKE_PRIVATE_KEY_PATH=/home/airflow/.ssh/snowflake_find_a_movie_rsa_key.p8
```

Snowflake authentication uses SSH key pairs stored in `secrets/` (not committed).
The key is volume-mounted into the container at `/home/airflow/.ssh/`.
