Como funciona

Um DAG, um relógio,
uma pista de decolagem.

01 · Definição
O DAG
Você descreve o workflow como código — nós são tasks, arestas são dependências.
02 · Scheduler
Decide quando
Lê o DAG, resolve agendamento e dependências, e enfileira as tasks prontas.
03 · Executor
Faz o trabalho
Executa cada task em um worker — cuida de retries, paralelismo e isolamento.
Cada task
executa um passo
Extrair Carregar Transformar Validar Publicar
E acima de tudo A UI — logs, retries, dependências, num só lugar.

Visão geral
A visão geral

Duas ferramentas.
Uma plataforma.

01

Airflow

→ Orquestração

Faz o pipeline rodar. Agenda, tenta de novo e expõe falhas antes que alguém precise perguntar.

02

dbt

→ Transformação

Dá forma aos dados. Testa, documenta e versiona cada modelo dentro do warehouse.

01 + 02

Juntos

→ Plataforma de dados confiável

A orquestração decide quando e em que ordem. O dbt decide o que os dados significam. Juntos: confiança em escala.

Construa o pipeline uma vez. Deixe o grafo se explicar sozinho. Fim.
